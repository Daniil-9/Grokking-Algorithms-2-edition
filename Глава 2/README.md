## Массивы и связанные списки

 - **Массивы** хранят элементы в памяти **вплотную** друг к другу. Это позволяет мгновенно прыгнуть к любому элементу по индексу (**быстрое чтение, O(1)**), но добавление или удаление требует переноса всех соседних элементов (**медленная запись, O(n)**).
   
 - **Связанные списки** разбрасывают элементы по всей памяти, где каждый элемент хранит «адрес» следующего. Благодаря этому в них очень легко вставлять новые данные (**быстрая запись, O(1)**), но чтобы найти нужный элемент, приходится перебирать всю цепочку с самого начала (**медленное чтение, O(n)**).
   
Короче: **массивы** - для чтения, **списки** - для вставки/удаления.  

`В Python **List** хоть и называется как "список", на самом деле это массив, т к данные лежат "сплошным блоком".`

`Каждый элемент связанного списка указывает на следующий элемент с помощью **указателей**. В каждом элементе выделяется дополнительная память для хранения этого адреса следующего элемента.`
   
Массивы используются чаще за исключением особых случаев. Например: если элементы слишком малы, то за счет доп памяти у связанных списко будет занято больше памяти, чем у массивов.

### Упражнения

2.1 Приложение для управления финансами. Записываете траты каждый день, а каждый месяц анализируете. Много записей и мало чтения. Что лучше?  
    Ответ: связанный список

2.2 Приложение для приема заказов. Должно хранить список заказов. Официанты добавляют новые в конец, а повар читает из начала. Какую структуру выбрать?  
    Ответ: связанный список

2.3 Facebook хранит список имен пользователей. При заходе кого-то на сайт, тот пытается найти его имя. Если он зарегистрирован, то вход разрешается. Этот БИНАРНЫЙ поиск происходит очень часто. Бинарный, т е произвольно ищем (а не последовательно). Массив/связанный список?  
     Ответ: массив

2.4 Пользователи также часто создают новый аккаунты. Мы используем массив для хранения списка пользователей. Недостатки массива для вставки? Бинарный поиск. Что произойдет при добавлении новых пользователей в массив?    
    Ответ: 1 - закладываем много доп места, что будет неэффективно, а также массив в целом не подходит для записей    
           2 - Бинарный поиск => нужен отсортированный массив => мы подем постоянно сортировать  

2.5 Гибридная форма: массив для каждой буквы имени и связанный список для самих имен внутри. Это быстрее или медленнее чистого массивы или связанного списка?  
    Ответ: медленнее поиск чем у чистого массива, а по всему остальному не хуже связанного списка.

## Сортировка выбором

**Алгоритм работы:** вы хотите отсортировать список по убыванию. Надо пройтись по списку и найти самый большой элемент. Этот элемент добавляем в новый список. Потом то же самое для следующих.

**Сложность:** O(n^2)

## Шпаргалка

 - Память компьютера напоминает огромный шкаф с ящиками.
 - Если вам потребуется сохранить нобор элементов, воспользуйтесь массивом или списком.
 - В массиве все элементы хранятся в памяти рядом друг с другом.
 - В связанном списке элементы распределяются в произвольных местах памяти, при этом в каждом элементе хранится адрес следующего элемента.
 - Массивы обеспечивают быстрое чтение.
 - Списки обеспечивают быструю вставку и удаление.
