# Глава 1. Знакомство с алгоритмами

## Бинарный поиск

### Определение
Это алгоритм, получающий на входе отсортированный список элементов. Если нужный нам элемент присутствует в списке, то он возвращает позицию этого элемента. В противном случае возвращает null.

### Принцип работы
Я загадал число от 1 до 100. Вы угадываете и используете как можно меньше попыток. При каждой попытке я отвечаю мало/много/угадал.

**Простой поиск** - вы начинаете перебирать все числа подряд (плохой способ).    
**Более эффективный поиск** - вы начинаете уже с 50. Таким образом исключаете половину чисел. Если я ответил много, тогда вы говорите уже 25 (новая половина)... и тд.

`100 -> 50 -> 25 -> 13 -> 7 -> 4 -> 2 -> 1`  
`=> 7 шагов`

`С бинарным поиском вы каждый раз говорите число в середине диапазона и исключаете половину оставшихся чисел.`    

**Бинарный поиск**: log2(n) шагов  
**Простой поиск**: n шагов  
Где **n** - количество элементов в списке.

### Примечания
1. Бинарный поиск работает только если список отсортирован.  
2. В Python массив = список.

    Ответ: 7

## Время выполнения
**Простой поиск** - линейное время выполнения (макс количество попыток = размеру списка)  
**Бинарный поиск** - логарифмическое время выполнения.  

| Простой поиск | Бинарный поиск |
|---------------|----------------|
| 100 элементов | 100 элементов  |
|-> 100 попыток |-> 7 попыток    |
| O(n) ->       | O(log n) ->    |
|линейное время | лог. время     |

## "O-большое"

`Время выполнения для бинарного и простого поиска растет с разной скоростью.`

`"O-большое" описывает скорость работы алгоритма. Если у нас есть список из n элементов, то простой поиск проверит каждый элемент, те выполнит n операций. Время выполнения "O-большое" имеет вид O(n). Оно не показывает скорость в секундах, а помогает сравнивать количество операций. Оно указывает, как быстро возрастает время выполнения алгоритма.`

### Запись
**O(n)**   
Где:  
O -> "O-большое"    
n -> Количество операций    

`"O-большое" определяет время выполнения в худщем случае.`

### Типичные примеры "O-большого"  
 **Скорость в порядке убывания**  
 - O(log n), или *логарифмическое время*. Пример: бинарный поиск.
 - O(n), или *линейное время*. Пример: простой поиск.
 - O(n * log n). Пример: эффективные алгоритмы сортировки (быстрая сортировка).  
 - O(n^2). Пример: медленные алгоритмы сортировки (сортировка выбором).
 - O(n!). Пример: очень медленные алгоритмы (задача о коммивояжере).

### Задача о коммивояжере

**Условие:** коммивояжер хочет объехать 5 городов - и сделать это за минимально возможное время.

**Решение:** перебрать все возможные комбинации порядка объезда городов. (для 5 элементов - 120 перестановок)
     
## Шпаргалка

 - Бинарный поиск работает намного быстрее простого.  
 - Время выполнения O(log n) быстрее чем O(n), а с увеличением размера списка, в котором происходит поиск значения, оно становится намного быстрее.  
 - Скорость алгоритмов НЕ измеряется в секундах.
 - Время выполнения алгоритма описывается ростом количества операций.
 - Время выполнения алгоритмов выражается как "O-большое".
